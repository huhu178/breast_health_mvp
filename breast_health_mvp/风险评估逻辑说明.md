# 风险评估逻辑说明

## 📊 风险评估方法概述

项目中存在**两种风险评估方法**，但目前已**不再使用**（已移除默认值）：

## 方法一：决策树风险评估（`decision_tree.py`）

### 评估逻辑

基于 **BI-RADS分级** 和 **家族史** 的简单规则匹配：

#### 风险等级判定规则

| 风险等级 | 判定条件 | 风险评分 | 立即行动 |
|---------|---------|---------|---------|
| **极高风险** | BI-RADS ≥ 5级<br>或<br>BI-RADS = 4级 + 有家族史 + 年龄 < 40岁 | 95分 | 72小时内完成穿刺活检<br>多学科会诊（MDT）评估<br>遗传咨询（BRCA基因检测） |
| **高风险** | BI-RADS = 4级<br>或<br>BI-RADS = 3级 + 有家族史 | 75分 | 2周内完成穿刺活检<br>建议遗传咨询 |
| **中风险** | BI-RADS = 3级 | 50分 | 3-6个月复查超声<br>密切观察结节变化 |
| **低风险** | BI-RADS ≤ 2级 | 20分 | 每年常规体检<br>保持健康生活方式 |

### 代码位置
- 文件：`backend/services/decision_tree.py`
- 函数：`_assess_risk()` (第468行)
- 规则定义：`_load_decision_rules()` 中的 `risk_assessment` 部分 (第54-124行)

## 方法二：综合风险评估（`llm_helpers.py`）

### 评估逻辑

基于 **4个维度** 的加权综合评分：

#### 评估维度

1. **家族史维度** (权重 30%)
   - 高危 (3分)：BRCA突变 或 一级亲属乳腺癌
   - 中危 (2分)：二级亲属乳腺癌
   - 低危 (1分)：无家族史

2. **疾病史维度** (权重 30%)
   - 高危 (3分)：浸润癌史 或 不典型增生
   - 中危 (2分)：纤维瘤 或 囊肿 或 炎症
   - 低危 (1分)：无相关疾病史

3. **影像学维度** (权重 25%)
   - 高危 (3分)：BI-RADS ≥ 4级 或 TNM分期 ≥ 4期
   - 中危 (2分)：BI-RADS = 3级 或 TNM分期 = 3期
   - 低危 (1分)：BI-RADS ≤ 2级 或 TNM分期 ≤ 2期

4. **分子标志物维度** (权重 15%)
   - 高危 (3分)：CA15-3 > 30
   - 中危 (2分)：CA15-3 > 25
   - 低危 (1分)：CA15-3 ≤ 25

#### 综合评分计算

```
总分 = 家族史评分 × 0.30 + 
       疾病史评分 × 0.30 + 
       影像学评分 × 0.25 + 
       分子标志物评分 × 0.15
```

#### 综合风险等级判定

- **高危**：总分 ≥ 2.5
- **中危**：1.8 ≤ 总分 < 2.5
- **低危**：总分 < 1.8

### 数据库规则表支持

如果数据库中存在 `risk_assessment_rules` 表，会从表中读取规则：
- 维度（dimension）：家族史/疾病史/影像学/分子标志物
- 风险等级（risk_level）：低危/中危/高危
- 判定标准（criteria）
- 监测频率（monitoring_frequency）
- 升级条件（upgrade_conditions）
- 权重百分比（weight_percentage）

如果表不存在，则使用硬编码逻辑（向后兼容）。

### 代码位置
- 文件：`backend/routes/llm_helpers.py`
- 函数：`calculate_comprehensive_risk()` (第2101行)
- 备用函数：`_calculate_risk_legacy()` (第2267行)

## 🔄 两种方法的区别

| 特性 | 决策树风险评估 | 综合风险评估 |
|------|--------------|-------------|
| **复杂度** | 简单规则匹配 | 多维度加权评分 |
| **评估维度** | 2个（BI-RADS + 家族史） | 4个（家族史+疾病史+影像学+分子标志物） |
| **灵活性** | 固定规则 | 支持数据库规则表配置 |
| **使用场景** | 决策树处理流程 | 独立风险评估API（已删除） |
| **风险等级** | 极高风险/高风险/中风险/低风险 | 高危/中危/低危 |
| **风险评分** | 20/50/75/95 | 1-3分（各维度）加权计算 |

```python
tree_result = decision_tree.process(patient_data, matched_knowledge)
risk_assessment = tree_result.get('risk_assessment', {})
risk_level = risk_assessment.get('risk_level', '低风险')
risk_score = risk_assessment.get('risk_score', 20)
```

### 方案2：使用综合风险评估
```python
from routes.llm_helpers import calculate_comprehensive_risk
comprehensive_risk, risk_scores, risk_details = calculate_comprehensive_risk(patient_data)
# comprehensive_risk: '低危'/'中危'/'高危'
# risk_scores: {'family_history': 1-3, 'disease_history': 1-3, ...}
# risk_details: 各维度详细信息
```

## 📌 注意事项

1. **决策树风险评估** 主要基于BI-RADS分级，适用于乳腺结节
2. **综合风险评估** 考虑了更多维度，但需要更多数据字段
3. 两种方法的风险等级命名不同（决策树用"极高风险/高风险/中风险/低风险"，综合评估用"高危/中危/低危"）
4. 目前两种方法都**未在报告生成流程中使用**，所有风险评估字段均为 `None`
