# 微信小程序测试指南

## 📋 准备工作

### 1. 确保后端服务已启动

在项目根目录下启动后端：

```bash
cd D:\1work\20251016\breast_health_mvp\backend
python app.py
```

看到以下输出说明启动成功：
```
✅ Loaded development config from config.py
✅ CORS enabled for development (localhost:5173, localhost:5000)
Flask API: http://localhost:5000
Vue Frontend: http://localhost:5173
Running on http://0.0.0.0:5000
```

**重要：** 确保看到 "Running on http://0.0.0.0:5000" 或类似输出，表示后端已启动。

### 2. 获取本机 IP 地址（用于跨机器访问）

**Windows PowerShell：**
```powershell
ipconfig
```
找到 "IPv4 地址"，例如：`192.168.1.100`

**如果后端和微信开发者工具在同一台电脑，可以使用：**
- `http://127.0.0.1:5000`（推荐）
- `http://localhost:5000`

## 🚀 在微信开发者工具中测试

### 步骤 1：打开微信开发者工具

1. 启动「微信开发者工具」
2. 点击左上角「+」按钮，创建新项目

### 步骤 2：导入项目

在弹出的「新建项目」对话框中：

1. **项目名称**：输入任意名称，如「健康问卷小程序」
2. **目录**：点击「目录」后面的「选择」按钮，选择 `D:\1work\20251016\breast_health_mvp\c` 文件夹
3. **AppID**：选择「测试号」或「无 AppID」
   - **推荐：测试号**（可以选择已有的测试号）
   - **或者：无 AppID**（仅用于本地开发测试）
4. **开发模式**：选择「小程序」
5. 点击「新建」按钮

**注意：** 
- 如果提示「目录下已存在项目配置文件」，点击「确定」即可
- 如果提示需要 AppID，可以选择「测试号」或使用「无 AppID」模式（点击「不使用云服务」）

### 步骤 3：配置后端地址

#### 方法 1：在页面中直接填写（最简单，推荐）

小程序首页顶部有「后端地址」输入框，直接填写：
- **本机测试**：`http://127.0.0.1:5000` 或 `http://localhost:5000`
- **局域网其他机器**：`http://192.168.x.x:5000`（替换为实际IP）

**提示：** 页面上的输入框会自动保存到全局数据，刷新后仍然有效。

#### 方法 2：在代码中修改（永久）

如果想永久修改，编辑 `app.js` 文件：

1. 在微信开发者工具中，点击左侧文件列表，找到 `app.js`
2. 修改第 18 行：

```javascript
globalData: {
  apiBaseUrl: 'http://127.0.0.1:5000'  // 本机后端地址
  // 或 'http://192.168.1.100:5000'    // 局域网其他机器
}
```

3. 保存文件（Ctrl+S），小程序会自动重新编译

### 步骤 4：测试流程

#### 4.1 填写基础信息

在首页填写：
- **后端地址**：`http://127.0.0.1:5000`（或你的IP地址）
- **手机号**：11位数字，如 `13800138000`
- **姓名**：任意文本，如 `测试用户`
- **结节类型**：选择一个类型（如「乳腺（breast）」）

#### 4.2 加载问卷题目

1. 确认后端地址已正确填写（如 `http://127.0.0.1:5000`）
2. 点击「加载问卷题目」按钮
3. 等待加载完成（按钮会显示"加载中..."）
4. **如果成功**：
   - 会显示问卷界面
   - 显示「第 1 / X 题」和当前分组标题
   - 显示第一题的标签和输入框
5. **如果失败**：
   - 会在页面底部显示错误信息
   - 查看「调试步骤」章节排查问题

#### 4.3 逐题填写

- **上一题**：返回上一题（第一题时禁用）
- **跳过**：跳过当前题目，自动跳到下一题
- **下一题**：保存当前答案，跳到下一题
- 根据题目类型：
  - **文本/数字**：直接在输入框中输入
  - **日期**：点击选择日期
  - **布尔值**：使用开关切换

#### 4.4 提交并生成报告

1. 填写完所有题目后（或跳过部分题目）
2. 点击「提交并生成报告」按钮
3. 等待生成完成（会显示"加载中..."）
4. 如果成功，会显示：
   - `risk_level`：风险等级
   - `report_code`：报告编号
   - `report_id`：报告ID
   - `patient_code`：患者编号

## 🔍 调试技巧

### 1. 查看控制台日志

**微信开发者工具：**
1. 点击底部「Console」标签（控制台）
2. 查看 `wx.request` 的请求日志
3. 如果有错误，会显示红色错误信息

**查看请求详情：**
- 点击「Network」标签（网络）
- 找到对应的请求，查看：
  - **Request URL**：请求地址是否正确
  - **Request Headers**：请求头信息
  - **Request Payload**：请求数据
  - **Response**：响应数据

**后端日志：**
- 在运行 `python app.py` 的终端窗口查看日志
- 查看是否有错误信息（红色错误或异常堆栈）

### 2. 测试后端接口是否可访问

在浏览器中直接访问以下地址，检查后端是否正常运行：

```
http://127.0.0.1:5000/api/health
```

如果返回 JSON 数据（包含 `success: true`），说明后端正常运行。

### 3. 常见错误排查

#### ❌ 无法加载 schema
**错误信息**：`网络请求失败` 或 `加载失败（HTTP XXX）`

**排查步骤：**
1. **检查后端是否运行**
   - 查看运行 `python app.py` 的终端窗口
   - 确认没有报错，且显示 "Running on http://0.0.0.0:5000"

2. **检查后端地址是否正确**
   - 确认小程序页面上的「后端地址」输入框填写正确
   - 本机测试：`http://127.0.0.1:5000` 或 `http://localhost:5000`
   - 不要有多余的空格或斜杠

3. **在浏览器中测试接口**
   - 打开浏览器，访问：`http://127.0.0.1:5000/api/miniprogram/questionnaire/schema?nodule_type=breast`
   - 如果返回 JSON 数据，说明接口正常
   - 如果显示错误页面，查看后端日志

4. **检查微信开发者工具的 Console**
   - 点击底部「Console」标签
   - 查看是否有红色错误信息
   - 常见的错误：
     - `request:fail url not in domain list`：**域名白名单问题，见下方解决方案**
     - `request:fail connect timeout`：后端地址无法访问，检查网络或防火墙

#### ❌ 域名白名单错误（`url not in domain list`）

**错误信息**：`http://127.0.0.1:5000 不在以下 request 合法域名列表中`

**原因**：微信小程序对网络请求有安全限制，只能访问配置在白名单中的域名。

**解决方案（开发阶段）**：

1. **在开发者工具中关闭域名校验（推荐）**
   - 点击右上角「详情」按钮
   - 切换到「本地设置」标签
   - ✅ **勾选「不校验合法域名、web-view（业务域名）、TLS 版本以及 HTTPS 证书」**
   - 点击「编译」按钮重新编译项目
   - 现在应该可以正常访问 `http://127.0.0.1:5000` 了

2. **如果方法 1 不行，检查是否使用了真机预览**
   - 真机预览时，必须配置合法域名
   - 开发阶段建议使用模拟器测试，或配置测试域名

**正式环境配置**（上线前必须）：
- 登录微信公众平台 → 开发 → 开发管理 → 开发设置
- 在「服务器域名」→「request合法域名」中添加你的 HTTPS 域名
- 注意：本地 IP（`127.0.0.1`）无法加入合法域名，正式环境必须使用 HTTPS 域名

#### ❌ 提交失败（HTTP 400/500）
**错误信息**：`请求失败（HTTP 400）` 或 `问卷数据不能为空`

**排查步骤：**
1. **检查必填字段**
   - 手机号：必须是11位数字（如 `13800138000`）
   - 姓名：不能为空
   - 后端地址：必须正确填写

2. **查看后端日志**
   - 在运行 `python app.py` 的终端窗口
   - 查找错误信息（通常是红色的异常堆栈）
   - 根据错误信息进行修复

3. **检查数据库连接**
   - 确认 `.env` 文件中的 `DATABASE_URL` 配置正确
   - 确认数据库服务正在运行

4. **查看 Network 标签**
   - 在微信开发者工具中，点击「Network」标签
   - 找到失败的请求
   - 查看 **Response** 中的错误信息

#### ❌ 小程序显示空白页面
**原因**：可能是代码编译错误

**解决方案：**
1. 查看「Console」标签，查看是否有编译错误
2. 点击右上角「编译」按钮，重新编译
3. 如果还有问题，检查代码是否有语法错误

#### ❌ 字段显示为英文
**原因**：后端 `FIELD_LABEL_MAP` 中缺少该字段的中文映射

**解决方案：**
1. **临时**：接受英文显示（不影响功能）
2. **永久**：在 `backend/routes/miniprogram_routes.py` 的 `FIELD_LABEL_MAP` 中添加映射

## 📱 真机调试（可选）

如果需要在实际手机上测试小程序：

### 1. 确保手机和电脑在同一局域网（同一 WiFi）

### 2. 获取电脑的局域网 IP

**Windows PowerShell：**
```powershell
ipconfig
```

找到 "IPv4 地址"，例如：`192.168.1.100`

### 3. 修改小程序后端地址

在微信开发者工具中：
1. 在小程序页面的「后端地址」输入框中填写电脑的局域网 IP：
   ```
   http://192.168.1.100:5000
   ```
   （替换为你的实际IP）

### 4. 在微信开发者工具中预览

1. 点击右上角「预览」按钮（眼睛图标）
2. 会生成一个二维码
3. 用微信扫描二维码
4. 在手机上打开小程序进行测试

**注意：**
- 真机调试时，后端地址必须使用局域网 IP（如 `192.168.1.100`），不能使用 `127.0.0.1`
- 确保电脑的防火墙允许端口 5000 的访问
- 确保手机和电脑连接的是同一个 WiFi 网络
